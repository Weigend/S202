# S202 User Guide

## Ziel

Dieses Dokument beschreibt S202 aus Anwendersicht. Es erklärt, was die Oberfläche zeigt, wie zentrale Begriffe im Tool verstanden werden und welche typischen Arbeitsschritte ein Benutzer ausführt.

## Geplanter Inhalt

1. Überblick über S202
2. Projekt oder Jar laden
3. Schichtendarstellung lesen
4. Pakete, Klassen und Levels verstehen
5. Abhängigkeiten ein- und ausblenden
6. Verletzungen interpretieren
7. What-if-Verschiebungen ausführen
8. CUTs markieren und prüfen
9. Ergebnisse für Refactoring-Planung nutzen

## Offene Punkte

- Screenshots der aktuellen UI ergänzen.
- Begriffe mit der Konzeptdokumentation abgleichen.
- Kurze Beispiele für typische Benutzerfragen sammeln.
